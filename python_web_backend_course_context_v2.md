# Python Web Backend Course -- Portable Context (v2)

## Основная цель обучения

Научиться **разрабатывать backend‑сервисы на FastAPI** на уровне,
достаточном для профессиональной разработки.

Для глубокого понимания используется инженерный путь изучения:

HTTP → Routing → API Architecture → WSGI → Flask internals → FastAPI

Таким образом:

-   **Конечная цель** --- уверенная разработка на FastAPI
-   **Промежуточная цель** --- понять, как работают веб‑фреймворки
    изнутри

Это позволяет использовать FastAPI **осознанно**, а не как «чёрный
ящик».

# Архитектура учебного проекта

Проект реализует **минимальный HTTP backend сервер**.

Структура:

server.py

Основной класс:

SimpleHandler(BaseHTTPRequestHandler)

Сервер запускается через HTTPServer.

Архитектурная цель --- постепенно превратить этот код в
**микро‑framework**.

------------------------------------------------------------------------

# Архитектура обработки запроса

HTTP Request ↓ BaseHTTPRequestHandler ↓ SimpleHandler ↓ Routing system ↓
Handler function ↓ JSON response

------------------------------------------------------------------------

# Реализованная система роутинга

В проекте реализована собственная система маршрутов.

Таблица маршрутов:

ROUTES

Пример:

(GET, /time) (GET, /hello) (GET, /users/`<user_id>`{=html}) (POST,
/operation) (GET, /echo) (POST, /echo)

Также используется функция:

build_routes()

которая автоматически формирует список доступных маршрутов.

------------------------------------------------------------------------

# Реализованные endpoints

## /hello

Метод:

GET

Пример запроса:

/hello?name=Ivan

Ответ:

{ "message": "Hello, Ivan" }

------------------------------------------------------------------------

## /time

Метод:

GET

Возвращает текущее серверное время.

------------------------------------------------------------------------

## /users/`<user_id>`{=html}

Метод:

GET

Пример:

/users/42

user_id извлекается из URL.

------------------------------------------------------------------------

## /operation

Метод:

POST

Тело запроса JSON:

{ "a": 10, "b": 20 }

Сервер выполняет операцию над параметрами.

------------------------------------------------------------------------

## /echo

Методы:

GET\
POST

Endpoint возвращает полученные данные.

Используется для тестирования API.
